# תרגיל 03: דף קטגוריה

**רמת קושי:** ⭐⭐⭐
**זמן משוער:** 60 דקות
**פוקוס:** Routing, filtering, list views

## מטרה

הוסף דף קטגוריה שמציג את כל הכתבות בקטגוריה מסוימת. למשל: לחיצה על תג "תרבות" → דף עם כל הכתבות בתרבות.

## רקע

יש לנו 7 קטגוריות מוגדרות ב-`src/data/categories.ts`: משחקים, תרבות, טכנולוגיה, פוליטיקה, ספורט, אורח חיים, דעות.

הניווט באפליקציה הוא דרך `Page` discriminated union ב-`App.tsx`. אין router אמיתי.

## משימות

1. **הוסף סוג עמוד חדש** ב-`App.tsx`:
   ```ts
   type Page = ... | { kind: "category"; slug: string };
   ```
2. **צור `CategoryPage.tsx`** ב-`src/app/components/`:
   - מקבל `slug: string` ו-`onExit`, `onOpenArticle`
   - טוען את הקטגוריה דרך `fetchCategory(slug)`
   - טוען את הכתבות דרך `listArticles({ category: category.name, page, pageSize: 10 })`
   - מציג: hero עם שם הקטגוריה (בצבע שלה), grid של כרטיסי כתבה
   - תומך ב-pagination (כפתורי קודם/הבא או "טען עוד")
3. **חבר את הניווט**:
   - ב-`MobileHome` או בכותרת הכתבה, כשלוחצים על שם הקטגוריה → `setPage({ kind: "category", slug })`
   - הוסף תפריט קטגוריות בכותרת
4. **טפל ב-edge cases**: אין כתבות בקטגוריה / קטגוריה לא נמצאה.

## רמזים

- **Plan mode** חיוני כאן — יש 3-4 קבצים שצריך לתאם ביניהם.
- **Sub-agent מועיל:** _"חקור איך עובד הניווט בין דפים ב-App.tsx ובאיזה רכיבים מתבצע onOpenArticle"_
- **שימור עיצוב:** קח השראה ממבנה הכרטיסים ב-`NewsSection.tsx` או `RecommendedSection.tsx`.
- **הפרדת אחריות:** רק `CategoryPage` יודע לטעון דאטה — `MobileHome` לא צריך לדעת כלום על קטגוריות.
