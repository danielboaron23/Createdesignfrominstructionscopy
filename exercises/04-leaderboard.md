# תרגיל 04: לוח מובילים (Leaderboard)

**רמת קושי:** ⭐⭐⭐
**זמן משוער:** 45 דקות
**פוקוס:** Data presentation, sorting, filtering

## מטרה

בנה דף "לוח מובילים" שמציג את המשתמשים המובילים לפי ניקוד מצטבר. אפשר פילטור לפי משחק ספציפי.

## רקע

ה-mock API חושף:
- `fetchLeaderboard(limit)` → מחזיר את ה-Top N לפי סך-הכל ניקוד
- `fetchUserScores(userId)` → היסטוריית משחקים של משתמש

יש 34 רשומות ניקוד דמה ב-`src/data/scores.ts` שמתפזרות על 18 משתמשים ו-9 משחקים.

## משימות

1. **הוסף `Page` חדש**: `{ kind: "leaderboard" }`.
2. **צור `LeaderboardPage.tsx`**:
   - טוען את הנתונים דרך `fetchLeaderboard(20)`
   - מציג טבלה: מקום, אווטאר, שם, סך ניקוד, משחקים, win rate
   - **3 הראשונים מודגשים** עם זהב/כסף/ארד 🥇🥈🥉
   - **המשתמש הנוכחי מודגש** ברקע שונה (גם אם לא ב-top 20)
3. **הוסף פילטר** לפי משחק:
   - dropdown עם כל המשחקים + "הכל"
   - בבחירת משחק, מחושב מחדש leaderboard רק לפי אותו משחק
   - לוגיקה: צריך לחשב מחדש מ-`SCORES` מסוננים — או חשוף `fetchLeaderboard({ gameId })` ב-mockApi
4. **חבר ניווט** מתוך הכותרת או דף הבית.

## רמזים

- **Sub-agent:** _"חקור איך scores מאורגנים ב-src/data/scores.ts ואיך לחשב leaderboard מסונן לפי gameId"_
- **חישוב win rate:** עיגול לאחוז שלם (`Math.round(rate * 100) + "%"`)
- **TypeScript:** שים לב ש-`gameId` הוא `GameId` discriminated union — השתמש בו לטיפוס בטוח
- אם בחרת להרחיב את `fetchLeaderboard` ב-mockApi, ודא שזה לא שובר קוד קיים
