# תרגיל 08: סקשן תגובות בכתבה

**רמת קושי:** ⭐⭐⭐⭐
**זמן משוער:** 75 דקות
**פוקוס:** Forms, optimistic UI, nested data, validation

## מטרה

הוסף סקשן תגובות לכתבה: רשימה של תגובות קיימות, אפשרות להגיב, ולהשיב לתגובה (nested replies).

## רקע

יש לנו 7 תגובות mock ב-`src/data/comments.ts` עבור הכתבה הראשונה. ה-mockApi חושף:
- `fetchComments(articleId)` — מקבל תגובות
- `postComment(comment)` — שולח תגובה חדשה

## משימות

1. **הוסף סקשן תגובות בתחתית `ArticlePage`**:
   - כותרת "תגובות (N)"
   - טוען דרך `fetchComments(article.id)`
2. **כל תגובה כוללת**:
   - אווטאר + שם משתמש + תאריך יחסי ("לפני 3 שעות")
   - טקסט התגובה
   - כפתור 👍 likes (mock - לא חייב לעבוד מול שרת)
   - כפתור "הגב" שפותח תיבה לתגובה מקוננת
3. **תגובות מקוננות (replies):**
   - הצג עם indent
   - השתמש ב-`parentId` כדי לבנות את העץ
4. **טופס הוספת תגובה**:
   - textarea + כפתור "פרסם"
   - ולידציה: לפחות 5 תווים, פחות מ-500
   - בלחיצה: `postComment(...)` + הוספה optimistic לרשימה
   - במקרה של שגיאה: rollback + הצגת שגיאה
5. **Empty state**: "אין עדיין תגובות. היו הראשונים!"

## רמזים

- **Optimistic UI:** הוסף את התגובה לסטייט מיד, ועדכן עם ה-id האמיתי אחרי שה-promise החזיר. אם נכשל — הסר.
- **תאריך יחסי:** `date-fns` כבר מותקן. השתמש ב-`formatDistanceToNow(date, { locale: he, addSuffix: true })`.
- **מבנה עץ:** קומפוננטה `<CommentNode>` רקורסיבית שמרנדרת replies.
- **Sub-agent:** _"מצא איך משתמשים בלוקאל ב-ArticlePage ומה המבנה של ה-body שלה"_
- **שמור על RTL** — ה-indent של replies צריך להיות בצד ימין.
