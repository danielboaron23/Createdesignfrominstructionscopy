# תרגיל 01: חילוץ דאטה מהמשחקים

**רמת קושי:** ⭐⭐
**זמן משוער:** 30 דקות
**פוקוס:** Refactoring, file organization, parallel tool calls

## מטרה

כל אחד מ-9 המשחקים מכיל מערך דאטה inline (מילים, דמויות, שירים וכו'). זה מקשה על תחזוקה ועל הוספת תוכן. המטרה: לחלץ את כל הדאטה לקבצים נפרדים תחת `src/data/games/` ולהשתמש בקבצים הקיימים שכבר הוכנו שם.

## רקע

תיקיית `src/data/games/` כבר מכילה דאטה מורחב לכל המשחקים. עכשיו צריך לחבר את המשחקים אליה ולהסיר את הדאטה ה-inline.

## משימות

1. **רפקטר את `FiveLettersGame.tsx`** כך שייבא את `FIVE_LETTER_WORDS` מ-`src/data/games/words.ts` במקום ה-array הפנימי.
2. **רפקטר את `WhoAmIGame.tsx`** לייבא `CHARACTERS` מ-`src/data/games/characters.ts`.
3. **רפקטר את `ConnectionsGame.tsx`** לייבא `CONNECTIONS_PUZZLES` ולבחור פאזל באקראי בכל משחק.
4. **רפקטר את שאר המשחקים** באותה צורה: WhichSong, Logical, TwentyQuestions, CoverStory, NotForQuoting.
5. **ודא שהמשחקים עדיין עובדים** — הרץ `npm run dev`.

## רמזים

- **Sub-agent מועיל:** השתמש ב-Explore agent כדי לאתר את כל המשחקים והבן את המבנה לפני שמתחילים: _"מצא את כל קבצי המשחקים תחת src/app/components/games/ ותאר את מבנה הדאטה ה-inline בכל אחד"_
- **Parallel:** אחרי שהבנת את המבנה, בקש מ-Claude לרפקטר 2-3 קבצים במקביל.
- **Plan mode** מומלץ מאוד כאן — תכנן לפני שאתה משנה 9 קבצים.
- שים לב לטיפוסים: ה-`WordEntry` החדש שונה מה-`string[]` הישן.
