# תרגיל 06: חיפוש בכתבות

**רמת קושי:** ⭐⭐⭐⭐
**זמן משוער:** 60 דקות
**פוקוס:** Debouncing, UX, instant search

## מטרה

הוסף חיפוש חי (instant search) בכתבות: שדה חיפוש בכותרת → תוצאות מתעדכנות כשמקלידים, עם הדגשה של הטקסט המחופש.

## רקע

`listArticles({ search })` כבר תומך בחיפוש בכותרת/דק/מחבר/תגים. צריך לבנות UX סביב זה.

## משימות

1. **הוסף שדה חיפוש בכותרת** (כפתור 🔍 שמרחיב לשדה input).
2. **דבאונס:** עדכן את השאילתה רק 300ms אחרי שהמשתמש הפסיק להקליד — לא בכל hit על המקלדת.
3. **תוצאות חיפוש דרופ-דאון:**
   - מציג 5 התוצאות הראשונות
   - כל תוצאה: תמונה זעירה, כותרת, קטגוריה
   - הדגשת הטקסט המחופש (`<mark>` או span עם רקע צהוב)
   - "ראה את כל ה-X התוצאות" → דף תוצאות מלא
4. **דף תוצאות מלא** (`{ kind: "search"; query: string }`):
   - מציג את כל התוצאות עם pagination
   - אפשרות לסנן לפי קטגוריה
5. **טיפול ב-edge cases:**
   - שאילתה ריקה → לא מציג תוצאות
   - אין תוצאות → "לא נמצאו כתבות עבור 'X'"
   - שגיאת רשת → ErrorBanner

## רמזים

- **useDebounce hook**: כתוב hook קצר שמחזיר את הערך אחרי `delay`:
  ```ts
  function useDebounce<T>(value: T, delay: number): T { ... }
  ```
- **AbortController**: בקריאת חיפוש קודמת — בטל אותה אם הגיעה חדשה. מונע race conditions.
- **Sub-agent:** _"חקור איך mockApi.listArticles מטפל בפרמטר search ואיך נראית הצורה של ArticleData"_
- **a11y:** השתמש ב-`role="combobox"`, חצי מקלדת לניווט בתוצאות.

- **היסטוריית חיפושים** (localStorage): הצג 5 חיפושים אחרונים כשהשדה ריק.

- **fuzzy search**: גם כשיש שגיאות הקלדה (התקן `fuse.js` או כתוב Levenshtein פשוט).
